## 🚀 Deployment Instructions

### Step 1: Create Render Services
```yaml
# Already configured in render.yaml:
1. syncveil-postgres (PostgreSQL)
2. syncveil-backend (Python/FastAPI)
3. syncveil-website (Static frontend)
```

### Step 2: Environment Variables for Backend

**Auto-Generated (Render manages these):**
- JWT_SECRET ← Auto-generated by Render
- VAULT_ENCRYPTION_KEY ← Auto-generated by Render
- DATABASE_URL ← Auto-linked from PostgreSQL service

**Manual Configuration (Required):**
```
ENV=production
CORS_ORIGINS=https://syncveil.software,https://www.syncveil.software
FRONTEND_URL=https://syncveil.software
```

**Optional Email Configuration:**
If using email verification, add:
```
EMAIL_ENABLED=true
BREVO_API_KEY=your_brevo_api_key
SMTP_FROM=noreply@syncveil.com
```

If not using email:
```
EMAIL_ENABLED=false
EMAIL_VERIFICATION_REQUIRED=false
```

### Step 3: Deploy

1. Push to main branch on GitHub
2. Render automatically builds and deploys:
   - Backend: Runs `pip install -r requirements.txt` → `alembic upgrade head` → `uvicorn start`
   - Frontend: Runs `npm install && npm run build` → serves from `dist/`
3. Database tables auto-create on first run

### Step 4: Verify Deployment

```bash
# Health check
curl https://syncveil-backend.onrender.com/health

# Expected response:
# {"status":"ok"}
```

## 🔒 Security Notes

1. **Never commit .env file** - Use Render's environment variable management
2. **JWT_SECRET must be 32+ characters** - Render auto-generates, but verify in production
3. **VAULT_ENCRYPTION_KEY must be 32+ characters** - Same as JWT_SECRET
4. **PostgreSQL automatically uses SSL** - Render enforces this
5. **CORS is strict** - Only specified domains allowed
6. **Rate limiting enabled** - Brute force attacks throttled

## 📊 User Data Storage

All user data is stored in PostgreSQL:
- **Users**: Email, password hash, verification status
- **Sessions**: Refresh tokens, device info, IP address, expiration
- **OTP Attempts**: For 2FA verification
- **Email Verifications**: One-time tokens
- **Login Logs**: Audit trail of all login attempts
- **Vault Files**: Encrypted user uploads (ephemeral storage)

## ⚠️ Known Limitations

1. **Vault Storage is Ephemeral**: Files stored in `/tmp/vault_storage` are lost on redeploy
   - Solution: Configure S3 or persistent storage for production
2. **Email Verification is Optional**: Can run without email service
   - Default: EMAIL_VERIFICATION_REQUIRED=false
3. **No Redis Cache**: Optional for rate limiting, but not required

## 🚨 Troubleshooting

### Blank Dashboard After Login
- Verify VITE_API_URL is set to backend domain
- Check browser console for CORS errors
- Verify JWT_SECRET is 32+ characters

### Email Not Sending
- Check EMAIL_ENABLED=true
- Verify BREVO_API_KEY and SMTP_FROM are set
- Check email settings in Render environment variables

### Database Connection Failed
- Verify DATABASE_URL is set
- Check PostgreSQL service is running
- Confirm password and database name match

### 404 on Frontend
- Verify static files are built to `frontend/dist`
- Check Render build command: `npm install && npm run build`
- Verify staticPublishPath is set to `dist`
