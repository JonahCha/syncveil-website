# Quick Start: Deploy SyncVeil to Render

## Prerequisites
- GitHub account with SyncVeil repository
- Render account (free tier available)

## 5-Minute Deployment

### Step 1: Add Environment Variables
In Render Dashboard for `syncveil-backend` service, add:

```
JWT_SECRET=<auto-generated by Render>
VAULT_ENCRYPTION_KEY=<auto-generated by Render>
DATABASE_URL=<auto-linked from PostgreSQL>
ENV=production
CORS_ORIGINS=https://syncveil.software,https://www.syncveil.software
FRONTEND_URL=https://syncveil.software
EMAIL_ENABLED=false
```

### Step 2: For Frontend Service
Set environment variables:
```
VITE_API_URL=https://syncveil-backend.onrender.com
NODE_VERSION=20.19.0
```

### Step 3: Deploy
Push to GitHub main branch - Render auto-deploys!

```bash
git add -A
git commit -m "Deploy to production"
git push origin main
```

### Step 4: Verify

**Health Check:**
```bash
curl https://syncveil-backend.onrender.com/health
# Response: {"status":"ok"}
```

**Test Signup:**
```bash
curl -X POST https://syncveil-backend.onrender.com/auth/signup \
  -H "Content-Type: application/json" \
  -d '{
    "email": "test@example.com",
    "password": "TestPassword123"
  }'
```

**Expected Response:**
```json
{
  "access_token": "eyJhbGc...",
  "refresh_token": "...",
  "user": {
    "id": "...",
    "email": "test@example.com",
    "email_verified": false
  }
}
```

## ⚠️ Important Notes

1. **Email Service**: Optional (EMAIL_ENABLED=false by default)
   - If not using email, users can login without email verification
   - To enable email, set BREVO_API_KEY and SMTP_FROM

2. **Vault Storage**: Ephemeral (lost on redeploy)
   - File encryption working correctly
   - For persistent storage, configure S3

3. **Free Tier Limits**: 
   - 100MB database, 512MB RAM
   - Sleeps after 15 minutes inactivity
   - Adequate for testing/demo

## 🆘 Troubleshooting

### API returns 500 errors
Check logs: `Render Dashboard → syncveil-backend → Logs`

### Dashboard shows blank after login
1. Check VITE_API_URL in frontend environment
2. Verify backend is running
3. Check browser console for CORS errors

### Database connection failed
1. Verify DATABASE_URL is set
2. Check PostgreSQL service is running
3. Confirm Alembic migration ran successfully
