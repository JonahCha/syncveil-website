### 1. Render Services (Already Configured)
```yaml
services:
  - syncveil-postgres (PostgreSQL free tier)
  - syncveil-backend (Python/FastAPI service)
  - syncveil-website (Static Vite frontend)
```

### 2. Deploy Process
```bash
# 1. Push to GitHub main branch
git push origin main

# 2. Render automatically:
#    a) Installs backend dependencies (pip install -r requirements.txt)
#    b) Runs migrations (alembic upgrade head)
#    c) Starts API (uvicorn app.main:app --host 0.0.0.0 --port $PORT)
#    d) Builds frontend (npm install && npm run build)
#    e) Serves static files (dist/)
```

### 3. Health Check
```bash
curl https://syncveil-backend.onrender.com/health
# Response: {"status":"ok"}
```

### 4. Test Authentication Flow
```bash
# 1. Signup
curl -X POST https://syncveil-backend.onrender.com/auth/signup \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"SecurePassword123"}'

# 2. Login
curl -X POST https://syncveil-backend.onrender.com/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"SecurePassword123"}'

# 3. Access Dashboard (with token)
curl https://syncveil-backend.onrender.com/api/dashboard \
  -H "Authorization: Bearer <access_token>"
```

## 🚨 Known Limitations & Solutions

### Limitation 1: Vault Storage is Ephemeral
**Issue**: Files stored in `/tmp/vault_storage` are lost on redeploy  
**Solution**: For production, configure S3 or persistent volume  
**Current Status**: Acceptable for MVP (files stored during session)

### Limitation 2: Email Service Optional
**Issue**: Email verification can be disabled  
**Solution**: Users can still login with OTP code via email (if EMAIL_ENABLED=true)  
**Current Status**: Working as designed

### Limitation 3: No Redis in Free Tier
**Issue**: Redis not available in Render free tier  
**Solution**: Rate limiting works without Redis (in-memory)  
**Current Status**: Acceptable for MVP
