# SyncVeil Frontend - React Migration

This is the React frontend for SyncVeil, a privacy-first platform with encrypted data storage and breach monitoring.

## Project Structure

```
src/
├── App.jsx                 # Main React application root with state management
├── index.jsx              # React entry point
├── styles.css             # CSS animations and utilities (backup reference)
└── components/
    ├── Navigation.jsx     # Top navigation bar with mobile menu
    ├── Footer.jsx        # Footer component
    ├── BreachMap.jsx     # Interactive world map visualization
    ├── NewsSection.jsx   # Breach news display
    └── views/
        ├── Home.jsx      # Landing page with hero, features, map, news
        ├── AuthChoice.jsx # Login/Signup forms
        ├── Dashboard.jsx  # Authenticated user dashboard with tabs
        └── InfoPage.jsx   # Info page placeholder

dist/                      # Production build output (generated by `npm run build`)
index.html                # React entry point with full SEO metadata
vite.config.js            # Vite build configuration
```

## Development

### Prerequisites
- Node.js 18+ and npm

### Installation
```bash
npm install
```

### Development Server
```bash
npm run dev
```
This starts the Vite development server on `http://localhost:5173` with hot module replacement (HMR).

### Production Build
```bash
npm run build
```
This creates optimized production build in the `dist/` folder with:
- Minified JavaScript and CSS
- Asset hashing for cache busting
- Optimized chunk splitting
- ~230KB total (minified + gzipped)

### Preview Production Build
```bash
npm run preview
```
This runs the production build locally on `http://localhost:4173`.

## Key Features

### Components

**App.jsx**
- Central state management: `currentView`, `isAuthenticated`
- View switching logic
- Authentication handlers (login, signup, logout)
- Dynamic navigation height calculation

**Navigation.jsx**
- Responsive navbar with mobile menu
- Mobile toggle with hamburger/X icons
- Navigation links with active states

**Home.jsx** (Landing Page)
- Hero section with gradient and animations
- Features grid (6 features with Lucide icons)
- Interactive breach map
- Breach news section
- Call-to-action section
- All animations: fade-in-up, fade-in-right, float, pulse

**BreachMap.jsx**
- Interactive world map using jsVectorMap
- 11 high-risk countries highlighted
- Dynamic tooltips showing breach statistics
- Responsive sizing
- Statistics cards: High-Risk Regions, Records Exposed, Attacks Detected

**Dashboard.jsx**
- Multi-tab interface: Overview, Vault, Monitor, Settings
- File upload with drag-drop
- Encryption progress simulation
- Responsive sidebar navigation
- Statistics cards with Lucide icons
- Logout functionality

**AuthChoice.jsx**
- Login and Signup forms with tab switching
- Email/password authentication
- Form validation UI

## SEO & Metadata

The application preserves all SEO-critical metadata:
- Meta description
- Open Graph tags (og:title, og:description, og:url, og:image)
- Twitter card tags
- Canonical URL
- Schema.org structured data:
  - Organization
  - WebSite
  - BreadcrumbList

These are defined in `index.html` and available to search engines.

## Styling

All styling is done through:
1. **Tailwind CSS** - Utility classes in JSX (via CDN)
2. **Embedded CSS** - Animations and complex layouts in `index.html <style>` tag
3. **Component-scoped styles** - Inline styles for dynamic values

**Key animations:**
- `animate-fade-in-up` - Bottom-to-top fade in
- `animate-fade-in-right` - Right-to-left fade in
- `animate-float` - Vertical floating motion
- `animate-pulse` - Pulsing opacity

## Responsive Design

Mobile-first responsive design with breakpoints:
- **Mobile**: < 640px (sidebar slides from left)
- **Tablet**: 640px - 1024px
- **Desktop**: 1024px+

## State Management

Uses React Hooks for state:
- `useState` - Component-level state (currentView, isAuthenticated, etc.)
- `useEffect` - Lifecycle and side effects (DOM updates, library initialization)
- No external state library (Redux/Zustand) for this complexity level

## Troubleshooting

### Dev server won't start
```bash
npm install  # Reinstall dependencies
npm run dev  # Try again
```

### Build fails
```bash
rm -rf node_modules dist  # Clean everything
npm install                # Reinstall
npm run build              # Rebuild
```

### HMR not working
- Check firewall settings
- Try `npm run dev -- --host`
- Clear browser cache
